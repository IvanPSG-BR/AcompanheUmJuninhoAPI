# Como Contribuir para o Projeto AcompanheUmJuninho API

Agradecemos o seu interesse em contribuir para o projeto AcompanheUmJuninho API! Suas contribuições são muito valiosas para nós. Este documento descreve as diretrizes para contribuir com o projeto.

## 1. Boas-vindas e Introdução

Bem-vindo(a) ao guia de contribuição do projeto AcompanheUmJuninho API. Estamos entusiasmados com a possibilidade de você se juntar à nossa comunidade e nos ajudar a melhorar este projeto. Seja você um desenvolvedor experiente ou alguém que está apenas começando, suas ideias e esforços são bem-vindos.

Este documento visa fornecer um caminho claro para suas contribuições, garantindo que o processo seja o mais suave e eficiente possível para todos.

## 2. Comportamento Esperado (Código de Conduta)

Para garantir um ambiente acolhedor e inclusivo para todos os contribuidores, esperamos que todos sigam nosso Código de Conduta. Ao participar deste projeto, você concorda em respeitar este código. Comportamentos inaceitáveis incluem, mas não se limitam a:

* Assédio de qualquer tipo.
* Comentários depreciativos ou ofensivos.
* Discurso de ódio.
* Qualquer forma de discriminação.

Por favor, seja respeitoso, atencioso e profissional em todas as suas interações. Se você presenciar ou for alvo de comportamento inaceitável, entre em contato com os mantenedores do projeto.

## 3. Como Contribuir

Existem várias maneiras de contribuir para o projeto AcompanheUmJuninho API:

* **Relatar Bugs:** Se você encontrar um bug, por favor, abra uma issue detalhada em nosso rastreador de issues. Inclua passos para reproduzir o bug, o comportamento esperado e o comportamento real.
* **Sugerir Recursos:** Tem uma ideia para um novo recurso ou melhoria? Abra uma issue para discutir sua proposta.
* **Escrever Código:** Contribua com código para corrigir bugs, implementar novos recursos ou melhorar a base de código existente.
* **Melhorar a Documentação:** Ajude-nos a manter a documentação atualizada e clara.

Antes de começar a escrever código, por favor, verifique as issues existentes para evitar trabalho duplicado e discuta suas ideias com os mantenedores do projeto.

## 4. Processo de Pull Request

Para garantir que suas contribuições sejam integradas de forma eficiente, siga o processo de Pull Request (PR) abaixo:

1. **Fork o Repositório:** Faça um fork do repositório principal para sua conta GitHub.
2. **Clone o Repositório:** Clone o seu fork para sua máquina local.

    ```bash
    git clone https://github.com/seu-usuario/acompanheumjuninhoapi.git
    ```

3. **Crie uma Nova Branch:** Crie uma branch com um nome descritivo para sua contribuição (ex: `feature/novo-recurso`, `bugfix/correcao-erro-login`).

    ```bash
    git checkout -b nome-da-sua-branch
    ```

4. **Faça Suas Alterações:** Implemente suas alterações, garantindo que o código siga as boas práticas e os padrões do projeto.
5. **Testes:** Certifique-se de que todos os testes existentes passem e adicione novos testes para cobrir suas alterações, se aplicável.
6. **Commit Suas Alterações:** Escreva mensagens de commit claras e concisas.

    ```bash
    git commit -m "feat: adiciona novo recurso de autenticação"
    ```

7. **Envie para o seu Fork:**

    ```bash
    git push origin nome-da-sua-branch
    ```

8. **Abra um Pull Request:** Vá para o repositório original no GitHub e abra um Pull Request da sua branch para a branch `main` (ou a branch de desenvolvimento apropriada).
    * **Título do PR:** Use um título claro e descritivo.
    * **Descrição do PR:** Forneça uma descrição detalhada das suas alterações, incluindo o problema que resolve, como foi resolvido e quaisquer considerações adicionais.
    * **Referencie Issues:** Se o seu PR resolver uma issue existente, referencie-a na descrição (ex: `Closes #123`).

## 5. Boas Práticas de Código

Para manter a qualidade e a consistência do código, siga estas boas práticas:

* **Padrões de Código:** Siga os padrões de código definidos no projeto (ESLint, Prettier).
* **Testes:** Escreva testes unitários e de integração para suas alterações.
* **Documentação:** Atualize a documentação relevante (comentários no código, arquivos README, etc.) para refletir suas alterações.
* **Clareza e Simplicidade:** Escreva código claro, conciso e fácil de entender.
* **Modularidade:** Mantenha as funções e módulos pequenos e com responsabilidades únicas.
* **Revisão de Código:** Esteja aberto(a) a feedback e sugestões durante o processo de revisão de código.
